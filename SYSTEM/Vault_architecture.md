# Архитектура хранилища

## Основные правила

- **Один файл = одна идея.** Никаких файлов «разное.md».
- **Не дублировать информацию** между заметками. Ссылаться через WikiLinks.
- **Каждый файл имеет frontmatter** по схеме [[Metadata_schema]].
- **Теги — по таксономии** [[Tag_taxonomy]].
- **Имена файлов — по соглашениям** [[Naming_conventions]].
- **Связи — по правилам** [[Linking_guidelines]].

## Структура папок

> Эта таблица — **человеко-читаемая документация** структуры волта. Машинно-управляемый источник правды для скилла `/new-note` — секция `folders` в `.claude/vault-manifest.yaml`. При добавлении новой папки обновляются оба файла одновременно (это делает `/new-note` автоматически).

| Папка | Содержимое | Пример |
| --- | --- | --- |
| `ANIME/` | Карточки аниме-тайтлов: сериалы, фильмы, OVA (`note_kind: anime`) | `ANIME/Demon_Slayer.md` |
| `PERSONS/` | Карточки людей из аниме-индустрии: режиссёры, сценаристы, композиторы, продюсеры, аниматоры (`note_kind: person`) | `PERSONS/Hayao_Miyazaki.md` |
| `CHARACTERS/` | Карточки вымышленных персонажей аниме и манги (`note_kind: character`) | `CHARACTERS/Kamado_Tanjiro.md` |
| `STUDIOS/` | Карточки анимационных студий-производителей (`note_kind: studio`) | `STUDIOS/Ufotable.md` |
| `MANGA/` | Карточки манги/произведений (`note_kind: manga`) | `MANGA/Berserk.md` |
| `SYSTEM/` | Мета-документы волта: схемы, правила, таксономии | `SYSTEM/Metadata_schema.md` |
| `attachments/` | Все вложения: изображения, PDF, аудио | — |

> **Заполни таблицу под свой волт.** Каждая папка = один топик-класс заметок (например, для технического волта — `CONCEPTS/`, `GUIDES/`, `PROJECTS/`; для домашней базы — `RECIPES/`, `TRAVEL/`, `BOOKS/`). При создании первой заметки скилл `/new-note` подскажет регистрацию новой папки.

## Куда класть новую заметку?

1. Это мета-документ о самом волте? → `SYSTEM/`
2. Скилл `/new-note` определит папку из `.claude/vault-manifest.yaml` → `folders`. Если ни одна не подходит — предложит создать новую и зарегистрирует в манифесте + в этой таблице.
3. Не использовать `/new-note`? → Пользователь руками определяет папку, добавляет запись в манифест и в эту таблицу.

## Антипаттерны

- Файлы «разное.md», «заметки.md», «temp.md» в корне.
- Дублирование одной и той же информации в нескольких заметках.
- Пустые файлы без frontmatter (либо заполнить stub, либо удалить).
- Заметки-сироты без единой ссылки.
- Папки, не зарегистрированные в манифесте → ломают routing `/new-note`.
