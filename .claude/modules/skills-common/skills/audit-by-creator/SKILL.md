# /audit-by-creator — Аудит карточки по её создающему скиллу

> [!important] Vision-gate
> Визуальный под-чек постера (скачивание/восприятие картинки) подчиняется секции `## Правила скачивания и валидации изображений` в `CLAUDE.md`. Перед фото-шагом пройди активную проверку мультимодальности (pre-flight probe) — не суди о постере по имени файла / URL / подписи. Остальной (механический + структурный) аудит от зрения не зависит.

Берёт существующую карточку и сверяет её с **тем самым создающим скиллом**, который такие карточки производит (`note_kind` → `/new-<kind>`). Текст создателя — это де-факто спецификация «как выглядит полная и правильная карточка»: frontmatter-поля, секции тела, форматы, реципрокные связи, постер, миниатюры галерей. Аудит **ре-применяет ожидания создателя** к готовой карточке, ловит недореализацию (типичную для слабых моделей), и попутно **оценивает качество модели-автора**, аппендя результат в накопительный леджер.

**Дополняет, не заменяет `/audit-note`.** `/audit-note` проверяет механику + контент + факты против **внешних** источников (governance-доки, веб). Этот скилл проверяет, что артефакт **структурно** такой, какой произвёл бы создатель, + оценку автора по структурной оси. Source-сверку фактов (`featured_in` vs внешний источник) НЕ делает — это домен `/audit-note`.

## Аргумент

`$ARGUMENTS`:
- **Имя/путь карточки** (обязательно). Если передано имя без пути — найти через Glob.
- `--no-images` / `-n` — пропустить визуальный под-чек постера (vision-gate). Механика постера (поле задано, файл валиден, embed-кэп) проверяется всё равно.
- `--report-only` — никаких правок: только отчёт + запись в леджер. Контент и `co_authored` не мутируются.
- `--no-ledger` — выполнить аудит без записи в леджер (ручной разовый прогон, когда телеметрия не нужна).

## Источник истины

1. **Создающий скилл** `note_kind` → `.claude/skills/new-<kind>/SKILL.md` — главная спека. Читать **целиком**.
2. **Транзитивно** — governance-доки, на которые ссылается создатель в своей секции «Источники правил» (минимум `SYSTEM/Metadata_schema.md`, `SYSTEM/Linking_guidelines.md`, `enums.yaml`/`SYSTEM/tag_taxonomy.yaml` если есть, и `references/tag-discipline.md` — правила тегирования + критерий приёма).
3. **`SYSTEM/spec_changelog.yaml`** (если есть) — даты ввода требований create-спеки, для epoch-conformance (`ledger-protocol.md §4.1`). Нет файла → пустой changelog → дрейфа нет.
4. **Протокол леджера** — `references/ledger-protocol.md` (рядом с этим скиллом): схема записи, формула score, epoch-реклассификация, нормализация ID, дедуп/контаминация, порядок `co_authored`. **Единый источник** для всех скиллов-писателей леджера, не дублировать его тело здесь.

## Ключевая идея: создатель — процедура, а не чеклист

Текст `/new-<kind>` содержит шаги **двух природ**, и скилл обязан их различать (иначе ре-ранит то, что не воспроизводимо):

| Природа | Пример | Что делает аудит |
| --- | --- | --- |
| **Артефактный инвариант** — проверяемо в готовой карточке | «`## Персонажи` = 2-колоночная таблица»; cover-embed `![[…\|300]]` сразу после H1; «миниатюра+WikiLink при наличии cover у цели»; «секция X только при `status: finished`»; pipe-escaping `[[X\|Y]]` | **СВЕРЯЕТ** с карточкой |
| **Процессный шаг** — про момент создания, не воспроизводится | Glob на существование; web search данных; «спроси multi-choice про enum»; «скачай постер» | **НЕ ре-ранит**; извлекает требуемое **конечное состояние** («постер присутствует и валиден») |

Это **reasoning-пас** («прочитать создателя → вывести артефактные инварианты → сверить»), не механическое исполнение чеклиста. Качество зависит от модели-судьи — поэтому `model: opus`.

## Алгоритм (единый проход: аудит → оценка → леджер → правки → авторство)

1. **Найти карточку**, прочитать. **Прочитать `co_authored`** (origin-автор — субъект оценки) и **`content_hash`** по контенту на входе (для дедупа/контаминации леджера). **Этот скилл `co_authored` НЕ перезаписывает** (`ledger-protocol.md §6` — аудит не авторствует): origin-автор должен пережить любую последовательность аудитов.
2. **`note_kind`** из frontmatter. Нет `note_kind` → сообщить «структурный аудит по создателю неприменим, запусти `/audit-note`», **стоп** (леджер не писать).
3. **Резолв создателя:** `note_kind` → `.claude/skills/new-<kind>/SKILL.md`. Скилла нет → «спеки-создателя для `<kind>` нет», **стоп**, **инварианты не выдумывать** (леджер не писать).
4. **Прочитать спеку:** создающий скилл целиком + транзитивно его governance-источники (см. «Источник истины»).
5. **Вывести артефактные инварианты** из текста создателя, отбрасывая процессные шаги (правила отбрасывания ниже), извлекая из них требуемое конечное состояние.
6. **Сверить карточку** с каждым инвариантом. Использовать **vault-index MCP-tools** вместо Grep, где применимо:
   - реципрокность межкарточных связей — vault-index MCP, tool `vault_asymmetric_links` (читает kind-пары из `vault-manifest.yaml::reciprocity_pairs`);
   - битые/дублирующиеся ссылки — `vault_broken_links`, `vault_duplicate_links`;
   - профиль/lint карточки (frontmatter, links_out) — `vault_note_profile`, `vault_lint`;
   - постер/изображения — `vault_image_status`;
   - basename-коллизия (если создатель требует глобальную уникальность имени) — `vault_duplicate_basenames`;
   - бэклинки — `vault_backlinks`.
7. **Классифицировать** каждую находку:
   - `OK` — инвариант соблюдён;
   - `MISSING` — обязательный артефакт отсутствует;
   - `WRONG_FORMAT` — артефакт есть, но не по спеке (формат/порядок/escaping);
   - `NEEDS_NETWORK_OR_VISION` — для вердикта нужен веб-fetch или визуальное суждение, которое нельзя решить вслепую. **Не угадывать.**
   - **Epoch-реклассификация** (`references/ledger-protocol.md §4.1`): прочитать `SYSTEM/spec_changelog.yaml` (если есть). Каждую `MISSING`/`WRONG_FORMAT`, чьё требование введено в спеку **позже** `created` карточки, перевести в класс **`SCHEMA_DRIFT`** — это **vault-долг**, не вина автора, **в score не входит**. Нет записи в changelog → требование считать пред-существовавшим (штрафуется). Нет/битый `created` → реклассификацию не применять, пометить «epoch unverifiable».
8. **Vision-gate** (только для визуального под-чека постера — «тот ли тайтл / не битый кадр»):
   - `--no-images`/`-n` задан → визуальный под-чек **пропустить** с пометкой, перейти к остальному.
   - Иначе **pre-flight probe ДО любого скачивания**: открыть через Read `.claude/assets/vision_probe.png`, вердикт по поведению Read. Вернул визуал → модель мультимодальна, можно делать визуальный под-чек постера. Ошибка / только метаданные → модель слепая → визуальный под-чек **пропустить** с пометкой `NEEDS_NETWORK_OR_VISION`, **ноль сетевых попыток**, без классификации по имени/URL.
   - **Механика постера** (`images.cover` задан, файл существует и валиден, cover-embed `![[…|300]]` сразу после H1) проверяется **всегда** — она от зрения не зависит, gate её не блокирует.
9. **Вычислить оценку 1–10** из находок по формуле `references/ledger-protocol.md §4` (штраф от базы: `−2` за MISSING, `−1` за WRONG_FORMAT; NEEDS_* и **SCHEMA_DRIFT без штрафа**; пол 1).
10. **Записать в леджер** (если не `--no-ledger`) по `references/ledger-protocol.md §2–§5`: `audit_type: structural`; `authored_model` = **нормализованный** (§3.1) origin-автор из `co_authored`, `authored_model_raw` = сырое значение; `judge_model` = нормализованная текущая модель; `score`, `findings` (вкл. `schema_drift`), `reasons`, `content_hash`, `invariants_total`; `fixes_applied` = применишь ли фиксы на шаге 11; `pristine` = нет ли более ранней записи по этой `note` с `fixes_applied=true` (§5.B). Соблюсти дедуп точных дублей (§5.A).
11. **Применить безопасные механические фиксы** (см. Fix-policy), если не `--report-only`.
12. **Метаданные ревизии — НЕ авторство** (`references/ledger-protocol.md §6`). **`co_authored` не трогать никогда** (аудит — судья, не автор; его след — в `ledger.judge_model`). Только если на шаге 11 реально были правки: `quality: draft`, `updated: <сегодня>`. Правок не было / `--report-only` → frontmatter не менять.
13. **Отчёт** (формат ниже).

### Правила отбрасывания «процедура → инвариант» (шаг 5)

- web search / web fetch / скачивание → не ре-ранить; конечное состояние («поле заполнено», «постер скачан») сверять как **наличие**; **числовую правдивость не перепроверять** (домен `/audit-note`).
- интерактивные вопросы (enum-расширение, выбор имени при коллизии) → не воспроизводить.
- pre-flight Glob на существование/уникальность при создании → отбросить (создание прошло); НО «коллизия basename **сейчас**» проверяема через `vault_duplicate_basenames`, если создатель требует глобально уникальное имя.
- reverse-нога cross-update создателя (обновить связанную карточку) → конечное состояние = «реципрокность замкнута» (`vault_asymmetric_links`), процесс обхода не воспроизводить. **НО сам марк-ап reverse-leg-артефакта — артефактный инвариант, а не процесс** (§22): строка субъекта в каст-таблице `featured_in`-карточки, sibling-буллет — это выход автора субъекта, и его разметка проверяема. Дефект разметки там (разорванная таблица `broken-table-row`, неэкранированный `\|` в `[[...]]`, отсутствие миниатюры при cover у цели) = дефект автора субъекта, классифицируется `WRONG_FORMAT` и штрафуется. `broken-table-row` (vault-index) детерминирует детект разорванной строки — не полагаться на глазомер.

### Пас консистентности, авторства и разлада (§23)

Сверх «артефакт присутствует?» — явный пас (судья выполняет на шаге 6–7, не штампует «секция есть → OK»):

- **(а) Кросс-поверхностная консистентность имени.** Имя субъекта должно быть согласовано между поверхностями: basename файла, H1, поля-имена (`name_*`), реверс-leg-упоминания в чужих карточках. Детерминированный бэкстоп — `name-surface-mismatch` (basename ≠ slug канонич. поля). Разлад имени (напр. западное имя в back-romanization, невалидный basename) = `WRONG_FORMAT` автора.
- **(б) Легитимность USER-ONLY-авторства.** Если governance объявляет секцию USER-ONLY (модель не заполняет — напр. семья `personal_score`/«Личный отзыв»), а на карточке она **не-стаб** при `co_authored`=модель и `quality≠verified` → вероятная фабрикация. Детерминированный бэкстоп — `user-only-fabricated` (heuristic-поток `vault_lint`). Верифицировать восприятием (нечёткий WARN), не штамповать.
- **(в) Разлад создатель↔governance.** Когда **сам создатель** (`/new-<kind>`) приглашает то, что governance запрещает (как `/new-character` приглашал «Личный отзыв», который семья USER-ONLY заполнять не должна), — **флагать разлад как vault-долг**, НЕ штрафовать автора (он conformed создателю) и НЕ штамповать «секция есть → OK». Это дефект спеки-создателя, а не модели. Отличать от §4.1 epoch-дрейфа: там требование появилось позже; здесь требование внутренне противоречиво. Класс — отдельная секция отчёта `CREATOR↔GOVERNANCE DISCORD`, в author-score не входит.

- **(г) Тег-консистентность (structural-класс, `ledger-protocol §4.2`).** Теги — structural-ось → судятся **здесь**, не в `/audit-note`. Детерминированные бэкстопы кормят судью (не штамповать «теги есть → OK»):
  - **canon-membership** — `non-canonical-tag` (`vault_lint`): тег вне канона (`SYSTEM/tag_taxonomy.yaml`) = author-defect формата (минтинг в обход admission-гейта / опечатка), `WRONG_FORMAT`. *(Severity правила — фазовая, см. `references/tag-discipline.md §5`: сейчас WARN, гейт-флип позже. Аудит штрафует разлад независимо от lint-severity.)*
  - **too-many-tags** (`vault_lint`, > `max_tags`) — `WRONG_FORMAT`.
  - **facet-пропагация (правило 9)** — `vault_tag_health` под-вывод `under_tag_discord`: субъект делит значение `tag_facet_fields` с сиблингами, но не несёт тематический тег, который у них есть → кандидат на дотегирование субъекта (артефакт автора субъекта, `WRONG_FORMAT`). Под-вывод `ghost`/`singletons` — **vault-level** гигиена, НЕ author-defect (не штрафовать автора за состояние таксономии).
  - Критерий приёма нового тега (D) и «широкая ось vs сущность-узкий шум» — **не structural** (нечёткое суждение), это heuristic-флаг `/audit-note`. Здесь — только детерминированная консистентность с каноном/сиблингами.

(Для `unmanaged`-создателя — см. «Краевые случаи»: там conformance-to-creator-as-is, разлад с governance не зона скилла. Пас (в) применяется к **managed**-создателям, где governance — общий источник.)

## Fix-policy

Дефолт — **авто-фикс безопасной механики** (self-heal); `--report-only` всё выключает.

**Авто-фиксить** (детерминированно, без потери пользовательского текста):
- отсутствующий обязательный заголовок-секция → пустой каркас с пометкой «заполнить»;
- pipe-escaping в таблицах (`[[X\|Y]]`);
- cover-cap `|300` на cover-embed;
- backfill миниатюр галереи при наличии cover у цели;
- тривиально-механическая конвертация формата секции, **не теряющая** текст (напр. bullet-список ↔ 2-колоночная таблица с теми же данными).

**НЕ фиксить молча** (флагать в отчёте):
- всё `NEEDS_NETWORK_OR_VISION`;
- добавление недостающего персонажа/сущности в каст (нужна source-сверка — домен `/audit-note`);
- фактический контент; смену enum; всё, что требует решения пользователя.

**`SCHEMA_DRIFT`** (epoch, §4.1) — vault-долг, не вина автора: можно **backfill'ить** механику (поле/секцию, введённые в спеку после `created`) тем же безопасным набором, но в score **не штрафовать** и в отчёте показывать отдельной секцией (это эволюция спеки, а не недоработка автора).

**Fix-scoping (§22) — чинить только артефакты субъекта-автора.** Дефект может физически лежать в **чужой** карточке (реверс-leg-строка субъекта в каст-таблице `featured_in`-аниме). Правило:
- Артефакт, **произведённый автором субъекта** (его реверс-leg-строка/буллет), — чинить можно, даже если он в чужом файле: это его выход, фикс безопасен.
- Со-локализованные дефекты, **принадлежащие автору чужой карточки** (другие строки той же таблицы, не про субъекта), — **НЕ чинить**, флагать как **vault-долг** (секция отчёта `VAULT-DEBT (чужие артефакты)`). Молчаливая правка чужого артефакта занизит будущий structural-score той карточки при её аудите — кросс-карточная контаминация (тот же механизм, что `pristine`). До аудита владельца — не трогать.

**Дисциплина fix-time (§23) — перечислить входящие ссылки перед мутацией референс-актива.** Перед переименованием / сменой расширения **референс-актива** (cover, общий вложенный файл) — сперва `vault_backlinks` на актив (индексирует и embeds) → собрать все inbound-референсы, обновить их в одной транзакции с переименованием. Иначе мутация рвёт миниатюры/постеры в ссылающихся карточках (self-inflicted `cover-ref-mismatch`). Лечит и пост-фактум-детект (`cover-ref-mismatch`), и процесс.

## Формат отчёта

```
## Аудит по создателю: <файл>  (note_kind → /new-<kind>)  | автор: <authored_model> → судья: <judge_model> | оценка: 6/10

### MISSING
- [section] Нет `## Студия и команда`
- [poster]  images.cover не задан, cover-embed отсутствует
### WRONG_FORMAT
- [## Персонажи] bullet вместо 2-колоночной таблицы
- [gallery] [[Sachiko|…]] без миниатюры при наличии cover у цели
### NEEDS_NETWORK_OR_VISION
- [poster] не подтверждаю визуально (модель слепая / --no-images)
### SCHEMA_DRIFT (vault-долг, в оценку не входит)
- [online_url] поле введено 2026-06-02 > created 2026-05-23 — автор не мог проставить
### CREATOR↔GOVERNANCE DISCORD (дефект спеки-создателя, в author-score не входит)
- [## Личный отзыв] создатель приглашает секцию, которую governance держит USER-ONLY — разлад спеки, не вина автора
### VAULT-DEBT (чужие артефакты, не чиним — ждут своего аудита)
- [Anime_X ## Персонажи] со-локализованная битая строка другого персонажа — флаг, не правка (контаминация score владельца)
### OK
- frontmatter полон; реципрокность anime↔character замкнута
### Применённые фиксы
- backfill миниатюр (Sachiko, Yumi); добавлен |300 к cover-embed
### Леджер
- записано: {audit_type=structural, authored=glm-4.6, judge=claude-opus-4.8, score=6, missing=2, wrong_format=3, schema_drift=1}  (или: пропущено — карточка не менялась с прошлой оценки / --no-ledger)
```

Всё OK → `<файл>: соответствует спеке /new-<kind>, дефектов нет` + запись в леджер со `score=10`.

## КРИТИЧНО

1. **Источник истины — текст создателя, не отдельный чеклист.** Не выдумывать инварианты и не держать вторую копию спеки. Нет создателя для kind → отказ, не угадывание.
2. **Различать артефакт и процесс.** Не ре-ранить web search / скачивание / интерактив. Из процессного шага извлекать конечное состояние, а не исполнять его.
3. **Аудит НЕ авторствует** (`ledger-protocol.md §6`). `co_authored` (origin-автор) **никогда** не перезаписывается этим скиллом — это субъект оценки, он обязан пережить любую последовательность аудитов. След судьи — только в `ledger.judge_model`. Это и чинит кросс-ран потерю автора (затирание `co_authored` одним аудитом ослепляло следующий).
4. **Леджер — побочный эффект, не отдельный прогон.** Никакого второго скилла ради оценки, никакой background-инфраструктуры. `judge_model` и `audit_type` — в каждой записи (фильтрация судьи/оси на запросе). Схема, формула, нормализация ID (§3.1), дедуп и контаминация-флаг `pristine` (§5) — только из `references/ledger-protocol.md`, не дублировать и не изобретать локально.
5. **Vision-gate узкий.** Активен только на визуальном под-чеке постера; механика постера и весь структурный аудит — без gate. Слепая модель / `--no-images` → пропуск под-чека с пометкой, ноль сетевых попыток. Не судить о постере по имени/URL.
6. **vault-index MCP-tools вместо Grep** для графовых/структурных проверок (реципрокность, ссылки, basename, image-status).
7. **NEEDS_NETWORK_OR_VISION не штрафуется** в score (не вина автора) и не фиксится молча.
8. **Только структурная ось.** Фактология, языковой мусор (CJK-вкрапления), качество текста — **НЕ эта ось** (`audit_type: structural`), это домен `/audit-note` (`audit_type: content`). Не чинить факты/текст под видом структурного аудита и не штрафовать за них в structural-score — они учитываются отдельной записью `/audit-note`. Полный портрет модели = обе оси, разными скиллами.
9. **Epoch — балл за качество, не за дрейф схемы** (`ledger-protocol.md §4.1`). Требование, введённое в спеку позже `created` карточки, → `SCHEMA_DRIFT` (vault-долг), **не штрафовать автора**: иначе старые baseline-карточки (сильные модели) систематически занижаются эволюцией спеки, и леджер врёт. Двусторонне: пред-существовавшее требование, отсутствующее, → честный author-defect.
10. **Reverse-leg марк-ап — артефакт автора, не процесс** (§22). Разметка реверс-leg-строки субъекта (в чужой каст-таблице) проверяется и штрафуется (`broken-table-row` детерминирует детект). Но **fix-scoping**: чинить только артефакты субъекта-автора; со-локализованные чужие дефекты — `VAULT-DEBT`-флаг, не молчаливая правка (контаминация чужого score).
11. **Консистентность ≠ присутствие** (§23). Сверять имя кросс-поверхностно (`name-surface-mismatch`), легитимность USER-ONLY-авторства (`user-only-fabricated`), **тег-консистентность** (canon-membership `non-canonical-tag`, facet-пропагация `under_tag_discord` — structural-класс, §4.2; критерий приёма нового тега — нечёткий, домен `/audit-note`), и **флагать разлад создатель↔governance** как дефект спеки — не штрафовать conformant-автора и не штамповать «секция/тег есть → OK». `ghost`/`singletons` — vault-гигиена, не author-defect.
12. **Fix-time: `vault_backlinks` перед мутацией референс-актива** (§23). Переименование/смена ext cover'а — сперва собрать inbound-референсы (embeds индексируются), обновить транзакционно. Иначе self-inflicted `cover-ref-mismatch`.

## Краевые случаи

- **`unmanaged` create-скилл** (пользователь правил создателя) — читать как есть: контракт = conformance-to-creator-**as-is**. Разлад создателя с governance — не зона этого скилла.
- **Карточка создана с `--no-images`** — эвристики «была ли так создана» нет → отсутствие постера = **WARNING** с оговоркой «возможно, --no-images; добавить через `/add-image`», не ERROR; в score жёстко не штрафовать.
- **Legacy-формат** — конформанс к **текущему** создателю; различия легитимно всплывут (это цель). В отчёте группировать, не давить объёмом.
- **Kind без создателя / `SYSTEM/`-заметки** — аудит неприменим, явный отказ, **леджер не писать**.
- **Прогон под не-референс-моделью** — записать с `judge_model = <та модель>`; вью отфильтрует. Но по `model: opus` судья и так должен быть Opus.
