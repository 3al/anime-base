# /verify — Пометить заметку как проверенную

Обновляет метаданные заметки после ревью.

## Аргументы

- `$1` — имя файла или путь (**опционально**). Если передано имя без пути, найти файл через Glob.
- `$2` — значение `quality` (по умолчанию `verified`). Допустимые: `verified`, `draft`.

**Если `$1` не указан:** определить целевой файл из контекста беседы — последняя заметка, которая была отредактирована, аудирована или обновлена скиллом. Если из контекста неоднозначно какой файл имеется в виду — отказать: «Укажите имя файла: `/verify <файл>`».

## Что делать

1. Найти файл по `$1` или из контекста (Glob если нужно). Если не найден — сообщить пользователю.
2. Прочитать файл.
3. Обновить в frontmatter **два поля** (НЕ трогать `co_authored` — его ставит скилл, который работал с контентом):
   - `quality` — установить в значение из `$2` (по умолчанию `verified`). Если поле уже есть — **заменить существующее значение in-place** (Edit: old_string → new_string на строке с `quality:`).
   - `updated` — установить в сегодняшнюю дату (формат `YYYY-MM-DD`). Если поле уже есть — **заменить существующее значение in-place**.
4. Если какого-то из этих полей нет в frontmatter — добавить после последнего существующего поля перед `---`.
5. Сообщить пользователю что обновлено.

## КРИТИЧНО: Не дублировать поля

**Перед каждым Edit убедись, что поле ещё не существует в frontmatter.** Алгоритм:
- Поле **уже есть** → заменить значение в существующей строке (например, `quality: draft` → `quality: verified`)
- Поле **отсутствует** → добавить новую строку

**НИКОГДА не добавлять новую строку с полем, которое уже присутствует.** Дублирование YAML-ключей ломает парсинг frontmatter. Если сомневаешься — перечитай файл и проверь наличие поля перед правкой.

## Пример

Вход: `/verify Some_Note`

Результат: в frontmatter файла `<path>/Some_Note.md`:
```yaml
quality: verified
updated: 2026-05-03
```

Вход: `/verify DRY draft`

Результат: в frontmatter файла найденного по имени `DRY`:
```yaml
quality: draft
updated: 2026-05-03
```

Вход: `/verify` (после `/audit-note Some_Note`)

Результат: применяется к `<path>/Some_Note.md` — последней отредактированной заметке в контексте.

Вход: `/verify` (в начале беседы, без предшествующих правок)

Результат: «Укажите имя файла: `/verify <файл>`».
