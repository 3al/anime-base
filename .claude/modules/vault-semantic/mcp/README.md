# vault-semantic (MCP server)

Semantic search L2 layer — see `docs/RAG_Architecture.md` for full spec.

**Status:** v0.1.0-scaffold. Skeleton only. Indexer, search, embedding,
chunking, lemmatizer modules contain stubs that raise `NotImplementedError`.
MCP server starts and registers placeholder tools that return
`{"status": "not_implemented"}`.

## Layout

```
mcp/
  pyproject.toml
  src/vault_semantic/
    __init__.py
    server.py           # MCP entry point (FastMCP)
    indexer.py          # stub
    search.py           # stub
    chunking.py         # stub
    lemmatizer.py       # stub
    embedding.py        # stub
    models.py           # ChunkResult / FilterSpec dataclasses (planned)
    schema/migrations/  # 001_init.sql (placeholder)
  .venv/                # generated by ops/install.mjs (gitignored)
  data/                 # generated at runtime (gitignored)
```

## Entry point

`python -m vault_semantic.server` — STDIO MCP server. Configured via
env: `VAULT_ROOT` (required), `HF_HOME` (HF cache dir, optional).
