# Frontmatter patterns — типизация полей note_kind

Каталог типовых паттернов полей frontmatter с YAML-примерами. Используется фазой 2 скилла `/add-note-kind` для формирования draft'а полей нового kind.

Каждый паттерн даёт: имя поля snake_case, тип, пример, когда применять.

---

## 1. `string` — свободный текст

Одна строка произвольного содержания.

```yaml
title_original: "鬼滅の刃"
location_name: "Лес за Звенигородом"
```

**Применять для:** имён, заголовков, кратких описаний, того что не повторяется между заметками.

---

## 2. `enum` — одно значение из закрытого списка

Закрытое множество допустимых значений. Список должен быть зафиксирован в `Metadata_schema.md`.

```yaml
edibility: poisonous
status: airing
forest_type: mixed
```

**Применять для:** категориальных свойств с малым числом вариантов (3–8). Эти поля обычно становятся обязательными тегами.

**В скилле:** обязательно перечислить все допустимые значения с описаниями. Без этого frontmatter будет «гнить» по мере добавления карточек.

---

## 3. `list[enum]` — список значений из закрытого списка

Несколько значений одного типа из закрытого множества.

```yaml
season:
  - summer
  - autumn
habitat:
  - deciduous-forest
  - meadow
genres:
  - action
  - supernatural
```

**Применять для:** свойств которые редко бывают единственными (биотопы, жанры, сезоны).

---

## 4. `list[string]` — список свободных строк

Список произвольных строк.

```yaml
aliases:
  - "Опенок луговой"
  - "Marasmius oreades"
mycorrhizal_partners:
  - birch
  - pine
```

**Применять для:** альтернативных имён, тэгов которые специфичны и не имеют смысла фиксировать в общем словаре.

**Разница с `list[enum]`:** если значения нужно стандартизировать (хочешь tag/filter — `list[enum]`), если просто перечисление — `list[string]`.

---

## 5. `date` — `YYYY-MM-DD`

```yaml
created: 2026-05-22
last_visited: 2025-09-14
release_date: 2019-04-06
```

**Применять для:** дат событий, ревизий, релизов.

**В скилле:** генерируемый `/new-<kind>` должен подставлять `created`/`updated` автоматически через `date +%Y-%m-%d` или текущую дату агента.

---

## 6. `dict[type→path]` — словарь типизированных файлов

Словарь, где ключ — тип/роль, а значение — путь к файлу в волте. Используется у mushroom-card для нескольких ракурсов фото.

```yaml
images:
  side: "attachments/Boletus_Edulis_side.jpg"
  top: "attachments/Boletus_Edulis_top.jpg"
  bottom: "attachments/Boletus_Edulis_bottom.jpg"
```

**Применять для:** когда у объекта есть **несколько типизированных вложений** (виды фото, документы разных категорий, аудиодорожки разных форматов).

**В скилле:** определить **закрытый набор ключей** (для anime: `poster`, `screenshot_1`, `screenshot_2`, `key_visual`). Не «свободный словарь» — иначе теряется смысл.

---

## 7. `list[wikilink]` — foreign keys на другие заметки

Список имён файлов (без `.md`) других заметок волта. Это **foreign keys**, по которым строится граф связей.

```yaml
lookalikes:
  - "Clitocybe_Rivulosa"
  - "Inocybe_Erubescens"
found_species:
  - "Boletus_Edulis"
  - "Cantharellus_Cibarius"
related_titles:
  - "Demon_Slayer_Mugen_Train"
```

**Применять для:** связей объект ↔ объект, особенно если их много или порядок важен.

**В Linking_guidelines** обычно появляется требование: «если перечислены здесь — должны быть WikiLinks в теле заметки».

---

## 8. `list[dict]` — список структурированных записей

Список словарей с фиксированным набором ключей. Применяется когда foreign key к другой заметке нужно дополнить **per-record метаданными** (количество, состояние, дата конкретной записи, и т.п.).

```yaml
found_species:
  - species: "Boletus_Edulis"
    count: 7
    condition: fresh
  - species: "Leccinum_Aurantiacum"
    count: 3
    condition: mature
```

Реальный пример — observation: запись похода фиксирует не только «какие виды нашёл», но и сколько, в каком состоянии. Плоский `list[wikilink]` потерял бы метаданные.

**Применять для:** «relation with attributes» — связь объект ↔ объект с дополнительными характеристиками этой конкретной связи. Альтернатива `list[wikilink]`, когда плоского списка недостаточно.

**В скилле:** строго зафиксировать **набор ключей** каждой записи, обязательность каждого, и допустимые значения для enum-ключей. Например:
- `species` (обязат., WikiLink) — имя файла карточки
- `count` (обязат., число) — количество
- `condition` (обязат., enum) — `young` / `fresh` / `mature` / `wormy` / `decaying`

**Конфликт с существующими полями:** перед тем как ввести `list[dict]` с тем же именем что уже есть в другом kind как `list[wikilink]` — явно обсудить с пользователем. Это **семантический конфликт**: одно имя поля, разный тип данных. Либо переименовать (`found_species` → `finds`), либо принять что у разных kind семантика отличается и закрепить это в Linking_guidelines.

---

## 9. `string` (single URL)

```yaml
mal_url: "https://myanimelist.net/anime/38000"
official_site: "https://kimetsu.com/"
```

**Применять для:** одной канонической ссылки на внешний ресурс. Если ссылок несколько (форумы, базы, обзоры) — лучше делать `list[string]` или просто секцию в теле заметки.

---

## 10. `number` / `range`

```yaml
episodes: 26
rating_mal: 8.51
year: 2019
duration_minutes: 24
```

**Применять для:** числовых характеристик с естественной сравнимостью.

**Замечание:** YAML может трактовать `1.0` как число с плавающей точкой и потерять trailing zero. Если важно — оборачивать в кавычки `"8.50"`.

---

## 11. `string` (single foreign key)

Когда связь one-to-one (один автор у книги, одна студия у аниме).

```yaml
studio: "ufotable"
author: "Murakami_Haruki"
```

**Применять для:** обязательных one-to-one связей. Если связь опциональна или часто отсутствует — лучше пропустить поле и оставить упоминание в теле.

---

## Антипаттерны

### A1. Слишком много опциональных полей

10+ опциональных полей превратят frontmatter в свалку. Если поле редко заполняется — оно должно быть в теле заметки, а не в frontmatter.

**Целиться:** 4–8 обязательных + 2–6 опциональных. Больше — пересмотреть, что действительно важно.

### A2. Свободные dict'ы

`dict[string→string]` без фиксированных ключей превращается в персональный шифр пользователя. Если решено сделать словарь — фиксировать **закрытое множество** ключей.

### A3. Дублирование с тегами

Если поле `genres: [action]` точно отражается в `tags: [action]` — это дубль. Лучше выбрать одно: либо field, либо tag. По правилу 6 тегирования (`.claude/skills/audit-by-creator/references/tag-discipline.md` — тег описывает содержание, ортогонально enum-полям): «Не дублировать то, что однозначно определяется другими полями frontmatter».

### A4. Произвольные числовые форматы

Не смешивать в одном поле `"24m"`, `24`, `"24 минуты"`. Выбрать единый формат (число, unit фиксированный в названии поля: `duration_minutes`).

## Шпаргалка по выбору

| Сценарий | Паттерн |
|---|---|
| Заголовок, имя | `string` |
| Дата создания/события | `date` |
| Категория из 3–8 вариантов | `enum` |
| Несколько категорий из словаря | `list[enum]` |
| Связь с другой заметкой (один) | `string` (foreign key) или WikiLink в теле |
| Связь с другими заметками (много, плоско) | `list[wikilink]` |
| Связь с другими заметками + per-record метаданные | `list[dict]` |
| Несколько типизированных файлов (фото-ракурсы) | `dict[type→path]` с фиксированными ключами |
| Числовая характеристика | `number` |
| Внешняя ссылка (одна) | `string` (URL) |
| Альтернативные имена | `list[string]` (всегда `aliases`) |
