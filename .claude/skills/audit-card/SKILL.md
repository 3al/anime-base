# /audit-card — Полный аудит карточки по обеим осям (оркестратор)

Гоняет на одной карточке **обе** оси леджера качества — `structural` (`/audit-by-creator`) и `content` (`/audit-note`) — так, чтобы **каждая** ось записала **чистый** (`pristine=true`) замер качества модели-автора, после чего применяет фиксы по обеим осям и закрывает карточку.

**Зачем нужен.** При наивном порядке «один аудит фиксит → второй видит `fixes_applied=true` → `pristine=false`» вторая ось систематически не попадает в сравнение моделей (`ledger-protocol.md §5.B`, кросс-осевая контаминация). Особенно больно при детерминированном workflow «по бэклогу всегда обе оси, structural первой» — content-ось набирает ноль pristine-данных. Эта обёртка снимает проблему **порядком вызовов с `--report-only`**, без правки managed-слоя.

**Это тонкий оркестратор.** Сам он не аудитит — всю работу делают два managed-скилла. Любое их ужесточение (новые инварианты, смысловой фактчек, lint-бэкстопы) обёртка наследует автоматически. Это **мост**: когда фреймворк сольёт обе оси в один проход, тело §«Алгоритм» схлопнется в один вызов.

## Аргумент

`$ARGUMENTS` = `<note> [--no-images]`.

- `<note>` — имя карточки без `.md`, относительный путь от корня волта, или абсолютный путь.
- `--no-images` / `-n` — проброс в `/audit-by-creator` (пропустить визуальный под-чек постера). На content-ось не влияет.

Примеры:
- `/audit-card CHARACTERS/Marlborough_Luculia`
- `/audit-card Hodgins_Claudia`
- `/audit-card CHARACTERS/Dietfried_Bougainvillea --no-images`

## Контракт, на который опирается обёртка (ledger-protocol)

Стабильные инварианты managed-скиллов — обёртка их **только использует**, не дублирует:

1. `--report-only` — **пишет ряд в леджер**, но **не мутирует** карточку (`audit-by-creator/SKILL.md`, `audit-note/SKILL.md`).
2. `--no-ledger` — выполняет аудит/фиксы **без** записи ряда.
3. Порядок каждого скилла: замер → запись в леджер → фикс (`ledger-protocol.md §6`). Замер всегда по контенту **на входе**.
4. `pristine=true` ⟺ в леджере нет более раннего ряда по этой `note` c `fixes_applied=true` (любая ось) (`ledger-protocol.md §5.B`).
5. Дедуп (`§5.A`): ряд пропускается, если совпали `note` + `judge_model` + `audit_type` + `content_hash`.

## Алгоритм

### 1. Резолв заметки

1. Снять флаг `--no-images`/`-n` из `$ARGUMENTS`, остаток = `<note>`.
2. Если `<note>` содержит `/` или `\` — нормализовать к относительному пути; убедиться, что файл существует.
3. Иначе — Glob `**/<note>.md`:
   - Один матч → использовать.
   - Несколько → структурированным multi-choice вопросом дать выбрать.
   - Ноль → ошибка.
4. Зафиксировать **резолвнутый путь** `P` — дальше передавать всем под-вызовам **именно его** (чтобы каждый под-скилл не переспрашивал резолв).

### 2. Предусловия

1. Прочитать frontmatter `P`: `note_kind`, `co_authored`, `created`.
2. **Structural-ось применима**, только если для `note_kind` существует создающий скилл `/new-<kind>` (его читает `/audit-by-creator`). Если создателя нет (например, системная заметка) — structural-ось пропустить, прогнать **только** content (шаг 4) и отметить в отчёте «structural N/A — нет /new-<kind>».
3. Запомнить факт: **изменилась ли карточка за прогон** (для флага края, шаг 6) — сравнивать контент `P` до шага 3 и после шага 5.5 (включая постер/реципрокность-доводку).

### 3. Ось A — structural-замер (report-only, без мутации)

Запустить **`/audit-by-creator --report-only P`** (+ `--no-images`, если задан).

- Пишет **row A**: `audit_type=structural`, `pristine=true`, `fixes_applied=false`. Это **чистый замер** structural-качества автора на нетронутом оригинале.
- Карточка **не меняется**.
- Удержать в контексте его **список механических фиксов** и находки — переиспользуются на шаге 5 (не передеривировать с нуля).

### 4. Ось B — content-замер + починка контента

Запустить **`/audit-note P`** (дефолт, без флагов).

- К этому моменту в леджере только row A (`fixes_applied=false`) → content-замер выходит **`pristine=true`**.
- Скилл чинит контент (проза/факты/язык/теги) и пишет **content-row**.
- В обычном случае (карточка битая по контенту) content-row несёт `fixes_applied=true` — это и есть маркер контаминации для будущих ре-аудитов.

### 5. Ось A — применить structural-фиксы (без второго ряда)

Запустить **`/audit-by-creator --no-ledger P`** (дефолт-режим минус леджер).

- Скилл ре-деривирует structural по **текущему** (уже контент-починенному) состоянию `P` и применяет безопасную механику. Ре-деривация против реального состояния **снимает риск stale line-edit** (камень 1): фиксы строятся по факту, а не по оригиналу шага 3.
- `--no-ledger` → второй structural-ряд **не пишется** (row A остаётся единственным structural-замером, и он чистый).
- Наследует fix-policy скилла (включая fix-scoping §22, когда приедет): чужие со-локализованные дефекты он флагнёт как vault-долг, а не починит.

### 5.5. Терминальная доводка: постер, reverse-leg propagation, реципрокность

Под-скиллы — **судьи**: они только **флагают** отсутствующий постер, рассинхрон reverse-leg-миниатюр и асимметрию связей, но терминально их **не достраивают** (постер/propagation = авторская работа `/add-images`/`/new-character`; замыкание связей = `/fix-links asymmetric`). Обёртка **терминальна** ([[feedback_audit_is_terminal_autochain_skills]]) → доводит всё до конечного состояния авто-чейном, без вопросов. Карточка не закрыта, пока постер на месте (или осознанно пропущен), миниатюра `P` стоит у всех ссылающихся карточек, и асимметрия с участием `P` = 0.

**(а) Постер.** Если structural-ось (шаг 3) отрепортила отсутствие постера (`images.cover` не задан / cover-embed нет / файла нет) — развести по флагу:

- `--no-images` / `-n` **передан** → осознанный обход vision-gate: постер не качать, пометить в отчёте «постер пропущен по `--no-images`», идти дальше. Probe не нужен.
- Флаг **не передан** → пройти **vision-probe** (канон `CLAUDE.md` → `## Правила скачивания и валидации изображений`: pre-flight probe через Read `.claude/assets/vision_probe.png`, вердикт по поведению Read):
  - **Мультимодальна** (Read вернул визуал) → авто-чейн `/add-images` на `P`: скачать → провалидировать (тип+размер) → Read-верифицировать (тот ли субъект) → уложить постер + прописать `images.cover` + embed `|300` под H1. Карточка обязана выйти **с постером**, после чего — обязательная propagation (под-пункт «б»).
  - **Не мультимодальна** (Read дал ошибку/метаданные) → постер **не трогать** (визуальная верификация невозможна, классификация по URL/имени запрещена), ноль сетевых попыток; пометить «постер пропущен — модель не мультимодальна; добавить через `/add-images` на мультимодальной модели».

  Почему в обёртке, а не в осях: `/audit-by-creator` — судья, постер не авторствует; «отдать карточку терминальной» — ответственность оркестратора. Без `--no-images` отсутствие постера = **незавершённость**, а не приемлемое состояние (иначе галерея персонажей в каст-таблице тайтла остаётся без миниатюры).

**(б) Reverse-leg propagation у ВСЕХ ссылающихся карточек** (систематически забывается). **Правило-источник истины — создающий скилл** (`/new-<kind>`, для персонажа — `/new-character` §7.5 «Reverse-leg: markup — субъект-артефакт, описание — карточки-хозяина»): при наличии у `P` cover'а его миниатюра обязана стоять во **всех** входящих ссылках. Обёртка не дублирует правило, а **исполняет** его здесь, потому что постер мог появиться в под-пункте «а» уже **после** структурного прохода (шаг 5), и `/audit-by-creator` его не увидел:

  1. Собрать веер входящих — vault-index MCP, tool `vault_backlinks(note=P)` (индексирует note-links и embeds), пройтись по **каждой** карточке, не только по каст-таблицам `featured_in`.
  2. **Миниатюра (субъект-артефакт `P`) — чинить.** Добавить `![[<P>_cover.<ext>|60]]` в каждый входящий буллет/ячейку без неё (привязана к cover'у `P`, §22-safe; pipe-escaping `\|` в таблицах). `updated` затронутых — на сегодня.
  3. **Описание-аннотация рядом — по тому, КТО её написал** (§7.5 пути i/ii, ground-truth `git blame`/`git log -L`, не дата): если аннотацию в чужой карточке вписал **создающий скилл `P`** (reverse-leg, путь ii) — это артефакт автора `P` → дефект (ложная роль / мутная фраза / факт) **чинить здесь** и скорить автору `P`; если её написал **автор самой карточки-хозяина** (путь i, в т.ч. проактивно про ещё-не-карточный `P`) — **флаг** на её `/audit-card`, не трогать (контаминация чужого score). Content-класс — домен `/audit-note`, не чинить факты под структурным видом.

**(в) Реципрокность связей** (систематически забывается → **явный шаг**, не дискреция). После всех правок прогнать reciprocity-проверку для kind карточки — vault-index MCP, tool `vault_asymmetric_links(sourceKind=<kind>, targetKind=<kind>)` по парам из `vault-manifest.yaml::reciprocity_pairs`, затрагивающим `note_kind` карточки (для `character` — `character↔character`; пары с `anime`/`manga` FK-backed и замыкаются своими reverse-leg). Любая асимметрия **с участием `P`** (как `source` или `target`) → авто-чейн `/fix-links asymmetric` (или точечное замыкание обратной ноги в затронутых карточках). Терминальное состояние = **0 асимметрии с участием `P`**; результат — в отчёт (в т.ч. «асимметрии не было»).

### 6. Проверка остаточного края (контаминация-guard)

Один квадрант ломает guard: **контент был чист (шаг 4 ничего не починил, `content-row.fixes_applied=false`), а структура битая (шаг 5 применил фиксы)**. Тогда дедуп (`§5.A`) душит ряд шага 5 (хеш входа == row A), и в леджере **нет** ни одного `fixes_applied=true` ряда, хотя карточка судьёй изменена.

- Детект: `карточка изменилась за прогон` **И** в леджере по `P` нет ни одного ряда с `fixes_applied=true`.
- Если сработало → **флаг в отчёте** (не авто-маркер): «⚠ Структура починена при чистом контенте; контаминация не записана в леджер. При будущем ре-аудите этой карточки выставить `pristine=false` вручную» (та же дисциплина, что [[feedback_audit_fixes_scoped_to_subject_author]] для Marlborough).
- Если не сработало (обычный случай) — ничего не делать.

### 7. Терминальный гейт закрытия + отчёт

**7.1. Детерминированный гейт закрытия (machine-checkable, НЕ дискреция).** Карточка `P` объявляется «закрытой» ТОЛЬКО после того, как детерминированный свип докажет чистоту. Прогнать ВСЕ проверки (vault-index MCP):
- tool `vault_lint(target=P)` (+ применимые ключи манифеста: `linkCap`, `coverField`, `nameSurfacePairs`, …) → требуется `summary.structural_green = true`;
- tool `vault_broken_links()` → НИ ОДНОЙ записи с `file = P` (битая WikiLink/embed на несуществующую цель — карточку ИЛИ файл обложки);
- tool `vault_duplicate_links()` → НИ ОДНОЙ записи с `file = P`;
- tool `vault_asymmetric_links(…)` по reciprocity-парам kind'а `P` → НИ ОДНОЙ записи, где `source` или `target` = `P`;
- постер: `images.cover` задан и файл на диске существует, ЛИБО осознанно пропущен (`--no-images` / не-мультимодальная модель).

Любой провал = карточка **НЕ закрыта**: починить (через соответствующую ось / под-чейн) и **перепрогнать весь свип** до зелёного. Гейт — это **пост-условие, а не процесс**: он ловит остаток независимо от того, что пропустили под-скиллы или предыдущие шаги (нельзя заставить модель «вызвать все tool'ы» — можно требовать, чтобы конечное состояние было машинно-доказуемо чистым). Зелёный `vault_lint` сам по себе НЕ закрывает: он НЕ покрывает broken/duplicate/asymmetric (это отдельные tool'ы; забыть их и довериться зелёному lint = классическая дыра, см. [[feedback_audit_must_run_link_tools]]). Результаты свипа — дословно в отчёт.

**7.2. Сводный отчёт:**

```
## /audit-card: <относительный путь>

  • замер: score N/10 (pristine=true, row A)   [или: N/A — нет /new-<kind>]
  • фиксы: <что применено на шаге 5>            [или: без правок]

Content (audit-note):
  • замер: score M/10 (pristine=<true|false>)
  • фиксы: <что применено на шаге 4>            [или: без правок]

  • reverse-leg propagation: <миниатюра P добавлена в N ссылающихся карточек (список) | веер уже синхронен>; описания: <M проверено, K починено (старше P), L флагнуто как vault-долг (младше P)>
```

## КРИТИЧНО

1. **Тонкий оркестратор — не аудитить руками.** Всю проверку/починку делают `/audit-by-creator` и `/audit-note`. Обёртка только задаёт порядок и флаги. Не дублировать их логику фиксов (иначе разойдётся с §22–24 при апдейте managed-слоя).
2. **Порядок строгий: A(report-only) → B(content) → A(no-ledger fix).** Он гарантирует оба `pristine=true`. Менять порядок нельзя — если structural починит до content-замера, content-ось станет `pristine=false` (ровно проблема, которую обёртка решает).
3. **`--report-only` на шаге 3 обязателен.** Без него шаг 3 замутит карточку → ось B потеряет pristine.
4. **Шаг 5 — `--no-ledger`.** Иначе второй structural-ряд (`pristine=false`) засорит леджер; чистый замер уже взят на шаге 3.
5. **Резолвить заметку один раз** (шаг 1), передавать резолвнутый путь под-вызовам — иначе тройной multi-choice на неоднозначном имени.
6. **`co_authored` никто не трогает** — оба под-скилла судьи, не авторы (`ledger-protocol.md §6`). Обёртка тем более.
7. **Это мост.** Когда фреймворк сольёт обе оси в один проход (Roadmap §17, вариант B) — заменить тело §«Алгоритм» (шаги 3–6) на один вызов слитого скилла. Остаточный край (шаг 6) тогда исчезнет by construction.
8. **Терминальность шире фиксов под-скиллов** (шаг 5.5). Под-скиллы — судьи: только **флагают** отсутствующий постер, рассинхрон reverse-leg-миниатюр и асимметрию связей, не достраивают. Обёртка доводит: постер через `/add-images` (если `--no-images` НЕ передан **и** vision-probe подтвердил мультимодальность); reverse-leg propagation (миниатюра `P` + аудит описаний у ВСЕХ ссылающихся карточек из `vault_backlinks`); реципрокность — через `/fix-links asymmetric`. Карточка **не закрыта**, пока постер на месте (или осознанно пропущен), миниатюра `P` у всех входящих, и асимметрия с участием `P` = 0. Это снимает три систематических промаха: (1) аудит флагал отсутствие постера вместо добавления, хотя флага `--no-images` не было; (2) при добавлении постера reverse-leg-миниатюра обновлялась только в каст-таблице тайтла, а веер ссылающихся карточек-персонажей оставался plain-WikiLink; (3) описание рядом с reverse-leg-миниатюрой (артефакт автора, scoping по дате создания) и реципрокность связей проверялись на дискреции и забывались.
9. **Закрытие гейтится пост-условием, не дисциплиной** (шаг 7.1). Карточка закрыта ТОЛЬКО при зелёном детерминированном свипе: `vault_lint` `structural_green` + `vault_broken_links`(P)=0 + `vault_duplicate_links`(P)=0 + `vault_asymmetric_links`(P)=0 + постер (или осознанный пропуск). Это лечит корневую дыру: мягкие прозовые шаги под-скиллов непроверяемы, а «зелёный lint» НЕ означает «ссылки целы» (lint не покрывает broken/duplicate/asymmetric). Инцидент 2026-06-07 (`Iris_Cannary`): пропущены `vault_broken_links`/`vault_duplicate_links` → 3 битые ссылки на несуществующие карточки+обложки, wrong-target (аниме вместо персонажа) и дубль дожили до «закрыто», поймал пользователь; обе оси леджера завышены (4/7→1/3). Гейт делает такой остаток непроходимым by construction. См. [[feedback_audit_must_run_link_tools]].
